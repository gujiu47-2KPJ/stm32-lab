# VSCode 烧录 STM32 配置说明

## 已配置的文件

1. `.vscode/launch.json` - 调试和烧录配置
2. `.vscode/tasks.json` - 构建任务配置

## 使用方法

### 方法1：调试并烧录（推荐）

1. **连接开发板**
   ```
   ST-Link → USB
   GND     → GND
   SWDIO   → PA13
   SWCLK   → PA14
   ```

2. **按 F5 启动调试**
   - 选择 "STM32 Debug (ST-Link)"
   - 程序会自动编译并烧录
   - 进入调试模式

3. **调试控制**
   - F5: 继续运行
   - F10: 单步跳过
   - F11: 单步进入
   - Shift+F11: 单步跳出
   - F9: 设置断点

---

### 方法2：仅烧录（不调试）

1. **按 F5 启动调试**
   - 选择 "STM32 Flash Only"
   - 程序会自动编译并烧录
   - 烧录完成后自动关闭

---

### 方法3：使用命令行烧录

#### 快速烧录

```bash
# 在 VSCode 终端中执行
cube-cmake --build build/Debug --target vscode_stm32
st-flash write build/Debug/vscode_stm32.elf 0x8000000
```

#### 清理并重新烧录

```bash
# 清理
cube-cmake --build build/Debug --target vscode_stm32 --clean-first

# 烧录
st-flash write build/Debug/vscode_stm32.elf 0x8000000
```

## 常见问题

### 1. 找不到 st-flash 命令

**解决方法**：
```bash
# 检查 ST-Link 工具路径
dir C:\Users\gujiu\AppData\Local\stm32cube\bundles\stm32cubemx\stlink-gui\bin

# 如果路径不同，修改 tasks.json 中的 stlinkPath
```

### 2. 连接失败

**检查项**：
- ST-Link 是否连接到 USB
- SWDIO 和 SWCLK 引脚是否正确连接
- 开发板是否供电

### 3. 编译错误

**解决方法**：
```bash
# 清理并重新编译
cube-cmake --build build/Debug --target vscode_stm32 --clean-first
```

## 配置文件说明

### launch.json

包含两个配置：
- **STM32 Debug (ST-Link)**: 调试模式，可以单步调试
- **STM32 Flash Only**: 仅烧录，不进入调试

### tasks.json

包含三个任务：
- **build**: 编译项目（默认任务）
- **clean**: 清理并重新编译
- **flash**: 编译并烧录

---

## 硬件连接

### ST-Link 连接

```
ST-Link 引脚   → STM32 引脚
---------------------------
SWDIO         → PA13
SWCLK         → PA14
GND           → GND
3.3V          → 3.3V (可选)
```

### 开发板供电

- 使用 USB 供电
- 或使用 ST-Link 的 3.3V 输出
